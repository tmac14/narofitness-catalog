## 3A — Backend preview PDF endpoint

### Request

```
POST /api/v1/catalogs/{catalog_id}/preview/pdf?cache_bust={optional}
```

| Param | Type | Notes |
|-------|------|-------|
| `catalog_id` | path UUID | Required |
| `cache_bust` | query string | Optional — cache invalidation / regeneration key |

**Body:** none

### Success response

| Field | Value |
|-------|-------|
| Status | `200` |
| Content-Type | `application/pdf` |
| Body | Raw PDF bytes |
| Header `X-Total-Pages` | Integer page count (PyMuPDF/fitz) |
| Header `X-Pdf-Engine` | Engine used (e.g. `playwright`, `weasyprint`) |
| Header `X-Preview-Generated-At` | ISO timestamp of generation |

### Error behavior (contract)

| Case | Expected |
|------|----------|
| Catalog not found | `404` — JSON `{ "detail": "..." }` |
| PDF engine unavailable | `503` — consistent with export path |
| Render failure | Error response; no partial PDF |

### Implementation rules

- Uses **real PDF/export rendering pipeline** via shared export helper (same family as `POST /export/pdf`).
- Context: `build_catalog_context(..., for_html_preview=False, api_base=settings.pdf_api_base)`.
- Page count: **PyMuPDF/fitz** on generated bytes → `X-Total-Pages`.
- Storage: writes under `data_dir/previews`; cleans files **older than 24h**.
- **Does not** create `CatalogExport` rows.
- **Does not** change `POST /api/v1/catalogs/{id}/export/pdf` behavior.

### Side effects

| Effect | Detail |
|--------|--------|
| Preview file write | Ephemeral cache under `data_dir/previews` |
| TTL cleanup | Previews older than 24h removed |
| CatalogExport | **None** |
| Jobs / ProcessRegistry | **None** |

## 3B — PDF.js paginated frontend viewer

### Components

| Component | Role |
|-----------|------|
| `PaginatedPreviewWorkspace` | Replaces legacy iframe preview |
| `PreviewPageNav` | Anterior / Siguiente / page indicator |
| `PdfPageCanvas` | Single-page render |
| `fetchCatalogPreviewPdf(catalogId, previewKey)` | Agent 4 API helper |

### Dependencies

- `pdfjs-dist` + PDF.js worker setup

### Behavior

| Rule | Contract |
|------|----------|
| Data source | `POST .../preview/pdf` blob + response headers |
| Display | **One real PDF page at a time** |
| Page count | From PDF.js document + `X-Total-Pages` header |
| Navigation | **Anterior**, **Siguiente**, **Página X de Y**, typed page number |
| Scroll | **No internal vertical scroll** inside page box |
| Reload | Resets to **page 1** after preview regeneration |
| Memory | Revokes blob URLs; destroys PDF documents on unmount/reload |
| Regenerate | **Regenerar vista previa** via `previewKey` / cache bust |

**Legacy iframe HTML:** retained only as **opt-in fallback** (Phase 3C) — not default.

## 3C — Fallback, stale, and export hardening

### Centralized Spanish copy

Lifecycle helper rules + user-facing strings for degraded/stale/fallback states.

### Degraded panel

Shown when PDF preview generation or PDF.js load fails:

- Primary: *“No se pudo generar la vista previa PDF.”*
- Secondary: engine / PDF.js / fetch-specific messages
- Action: **Reintentar**

### Opt-in HTML fallback

| Rule | Copy / behavior |
|------|-----------------|
| Trigger | Explicit user click only — **“Vista aproximada HTML”** |
| Warning | *“La paginación puede no coincidir con el PDF exportado.”* |
| Page count | **No fake page count** in fallback mode |
| Scroll | Continuous HTML scroll (approximate) |

### Stale preview lifecycle

| Event | Stale behavior |
|-------|----------------|
| `refreshPreview` start | **Does not** clear stale flag |
| PDF preview `onLoad` success | Clears stale **only after** successful PDF readiness |
| PDF preview `onError` | **Preserves** stale flag |
| Export-after-preview | **Aborts** if preview regeneration fails |

### Stale copy

- *“La vista previa PDF está desactualizada…”*
- *“Regenera la vista previa para ver la paginación final.”*

### Export

- **Unchanged:** `POST /api/v1/catalogs/{id}/export/pdf`
- Export remains **authoritative** download path; preview is non-persistent cache.

## Known limitations

| # | Limitation |
|---|------------|
| 1 | First preview generation can take **several seconds** (synchronous) |
| 2 | Preview generation is **synchronous** — no background job system |
| 3 | No ProcessRegistry / queue / status-bar job progress for preview |
| 4 | HTML fallback is **approximate** — continuous scroll; pagination may not match PDF |
| 5 | Preview/export **pixel parity** still requires **manual QA** |
| 6 | **PDF.js bundle size** increase (`pdfjs-dist` + worker) |
| 7 | Large catalogues may feel slow on first Regenerar |
